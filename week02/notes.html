const and let are better to use because their scoping is better.

Non-primitive values are by reference, so assignment can mutate the original data

    const c = { value: 1 };
    let d = c; // c.value = 1, d.value = 1
    d.value = 2; // c.value = 2, d.value = 2

Javascript will attempt to coerce types when doing comparisons. Keep this in mind when determining whether to use == vs ===. JS is weakly-typed

"lazy evaluation" will prevent JS from checking all conditions if it's already determined to be a failure.

Arrays, Logic, and Loops

Linear, ordered, RA

delete keyword gets rid of item at index, but does not minimize the array

    const arr = [1,2,3,4,5,6,7,8,9];
    delete arr[4]
    >> [ 1, 2, 3, 4, undefined, 6, 7, 8, 9 ]

Can destructure using the order and arrays

    const [x,y] = [1,2];
    >> x === 1
    >> y === 2

When using .sort, remember that numbers are sorted alphabetically

Functions

Function declaration:

    function hello(){
      console.log('Hello World!');
    }

Function expression:

    const goodbye = function(){
      console.log('Goodbye World!');
    };

Functions with default parameters:

    function hello(name='World') {
      console.log(`Hello ${name}!);
    }

Variables that are declared using the var keyword are hoisted in addition to functions! They will be undefined until given a value.
