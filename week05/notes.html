Testing and Debugging

Sources of errors

  • System Error
  • Programmer Error
  • User Error

An exception is an error.

A stack trace can be used in order to see the order of function calls that lead to an error.

Uses of strict mode:

  • cannot assign a nondeclared variable
  • cannot assign read-only globals

Strict mode can be used at a functional level

alert is garbage; console.log is life.

debugger is a tool that will allow you to step through the code line by line.

We can use the throw keyword in order to raise an error. We can Either throw an object for an error or create an error using the new Error() syntax.

We can use try, catch, and finally in order to handle errors.

Jest is a common testing framework. It can be ued in order to write tests quickly.
