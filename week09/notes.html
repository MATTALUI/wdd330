The Window Object

The BOM makes sense when your javascript is running in the context of a browser.

Nondeclared variables automatically egt added to the high level global context, the window. The same is true of JS functions.

The window has access to some dialog functions. (Note: They are very annoying.)

  • alert
  • confirm
  • prompt

window.location can be used to update pieces of navigation

You can use window.open to open a new browser window. This will be useful for popping up the websites of our sponsors!

You can create custom cookies.

    const cookies = document.cookie.split("; ");
    for (crumb of cookies){
      const [key,value] = crumb.split("=");
      console.log(`The value of ${key} is ${value}`);
    }

We can use the window.requestAnimationFrame recursively in order to do some animation programatically. Its FPS is usually about 60, but it's optimized depending on the browser/device. Although it's followed up by suggeting that we just use CSS instead.

HTML5 APIs

The data- attribue can be used to add whatever data we want to HTML elements.

Use localstorage to store key-value pairs in the browser.

You can add an event listener to the local storage! How nifty.

Javascript can access the user's geolocation.

Web workers can be used in ordre to run a very intensive process without fear of causing the browser to think that a script has become unresponsive. You can also communicate with them via messages.

    const worker = new Worker('task.js');
    worker.postMessage('Hello');
    worker.addEventListener('message', (event) => {
      console.log(event.data);
    }, false);

Websockets can be made using a URL

    const URL = 'wss://echo.websocket.org/';
    const connection = new WebSocket(URL);
    connection.addEventListener('open', () => {
      output('CONNECTED');
    }, false);

Youre scripts can use the Notification API in order to notify users.

Some APIs on the bleeding edge can be dangerous. We've even gotten rid of some that caused battery snitching.
